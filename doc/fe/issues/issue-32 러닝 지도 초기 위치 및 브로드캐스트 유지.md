# issue-32 러닝 지도 초기 위치 및 브로드캐스트 유지

> **작성일**: 2026-07-02  
> **관련 기능**: `feat #19 러닝 경로 렌더링 개선`  
> **브랜치**: `feat/19-route-rendering`

## 문제

- 온보딩에서 위치 권한을 승인해도 러닝 탭 첫 진입 시 지도가 즉시 현재 위치 중심으로 잡히지 않는다.
- `RunningView`를 벗어나면 `RealtimeDBService.stopBroadcast`가 호출되어 다른 탭 사용 중 친구들에게 내 위치가 보이지 않는다.
- 음악 재생 중에도 러닝 화면에서 현재 곡 상태를 바로 확인할 수 있는 UI가 없어 회귀처럼 느껴진다.

## 재현 흐름

1. 신규 설치 후 로그인/회원가입을 완료한다.
2. 온보딩 위치 권한에서 허용을 선택한다.
3. 메인 탭 진입 후 러닝 탭을 연다.
4. 러닝을 시작하지 않은 상태에서 지도 초기 위치를 확인한다.
5. 다른 탭으로 이동한 뒤 다른 계정 또는 친구 디바이스에서 내 위치 노출 여부를 확인한다.
6. Apple Music 재생 중 러닝 탭에서 현재 곡 UI 노출 여부를 확인한다.

## 기대 결과

- 위치 권한 승인 사용자는 러닝 탭 첫 진입 시 바로 현재 위치 중심 지도를 본다.
- 앱 사용 중 탭 이동만으로는 `activeRunners/{uid}`가 삭제되지 않는다.
- 러닝 화면에서 현재 재생 곡을 즉시 확인하고 음악 시트로 진입할 수 있다.

## 수정 방향

- 위치 수집은 앱 레벨에서 유지하고, 러닝 경로 기록만 러닝 세션 상태에 맞춰 제어한다.
- 위치 브로드캐스트 시작/중지는 탭 루트 수명주기에서 관리한다.
- 러닝 화면에 현재 위치 마커 가시성을 보완하고, 현재 곡 미니 UI를 추가한다.
