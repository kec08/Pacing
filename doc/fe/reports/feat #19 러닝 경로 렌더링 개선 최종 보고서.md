## 구현 요약

러닝 시작 전에도 지도에 경로가 누적되던 흐름을 정리하고, 러닝 중에만 경로가 기록되도록 위치 추적과 경로 기록을 분리했다.  
추가로 지도 탭 첫 진입 시 위치 권한 요청과 현재 위치 포커싱이 바로 동작하도록 수정했고, 앱 사용 중 다른 탭으로 이동해도 내 위치 브로드캐스트가 유지되도록 구조를 보완했다.

내 위치 핀은 다시 친구 핀과 동일한 스타일로 현재 듣는 노래 정보를 표시하도록 복구했고, 시작 버튼 상단에 별도로 노출되던 노래 카드 UI는 사용자 요청에 따라 제거했다.  
또한 `locationManager:didFailWithError:` delegate 누락으로 발생하던 런타임 크래시와, 초기 위치 업데이트가 과도하게 필터링되어 내 좌표가 보이지 않던 문제를 함께 수정했다.

## 구현된 기능 목록

- [x] 러닝 시작 전 경로 비노출 및 경로 기록 상태 분리
- [x] 일시정지 후 재개 시 기존 경로 유지
- [x] 경로 라인 그라데이션 및 렌더링 개선 유지
- [x] 지도 탭 첫 진입 시 위치 권한 요청 및 현재 위치 추적 시작
- [x] 초기 위치 업데이트 시 현재 좌표를 빠르게 지도에 반영
- [x] 앱 사용 중 탭 이동 시 내 위치 브로드캐스트 유지
- [x] 내 위치 핀의 현재 재생 곡 정보 표시 복구
- [x] 시작 버튼 상단 별도 노래 카드 UI 제거
- [x] `locationManager:didFailWithError:` delegate 추가로 크래시 방지

## 알려진 제한사항

- 위치 표시는 초기 응답성을 위해 완화된 정확도 좌표도 반영하지만, 러닝 경로 기록은 기존처럼 정확도 기준을 통과한 좌표만 저장한다.
- 앱 사용 중 탭 이동에서는 위치 브로드캐스트가 유지되지만, 앱 종료 또는 브로드캐스트 중지 로직이 호출되는 경우에는 기존 종료 정책을 따른다.
- MusicKit/MediaPlayer 동작은 실기기 환경 영향을 받으므로, 음악 정보 노출 확인은 실기기 QA 기준으로 본다.
