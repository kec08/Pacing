## 구현 요약

친구/홈/마이 화면의 세부 UX를 정리해 정보 우선순위와 화면 일관성을 맞췄다.  
추천 친구는 기존 가로 카드형에서 친구 목록과 같은 세로 리스트형으로 바꿨고, 홈의 `같이 들은 러너`는 실제 최근 세션을 읽어 상대 닉네임과 같이 들은 시점을 표시하도록 연결했다.

마이 페이지는 프로필 헤더에서 신체 정보를 제거하고 친구 화면과 같은 활동 상태 텍스트 중심으로 정리했다.  
추가로 `프로필 수정` 버튼을 실제 편집 시트로 연결해 프로필 사진, 닉네임, 키, 몸무게, 나이를 수정할 수 있게 했고, 지도의 내 위치/주변 러너 핀도 프로필 사진이 있으면 사진으로 표시되도록 확장했다.

- [x] `ListenSession` 상대 닉네임 계산 로직 보정

## 계획서 대비 변경 사항

| 항목 | 계획 | 실제 구현 | 사유 |
|------|------|-----------|------|
| 추천 친구 UI | 세로 리스트형 전환 | 친구 목록과 동일한 정보 구조의 리스트형 행으로 정리 | 친구/추천 정보 밀도를 맞추기 위해 |
| 홈 같이 듣기 | 최근 세션 조회 추가 | Realtime DB `listenSessions`에서 host/guest 양방향 최근 세션 병합 조회 | 기존 ViewModel이 빈 배열 고정 상태라 실제 데이터 연결 필요 |
| 프로필 수정 | 편집 화면 또는 시트 검토 | 별도 `ProfileEditView` 시트 추가 | 온보딩 단계형 화면을 그대로 재사용하기보다 수정 전용 폼이 안전 |
| 지도 프로필 사진 | 계획서 범위 없음 | 지도의 내 핀/주변 러너 핀에 프로필 사진 우선 표시 추가 | 개발 중 추가 요청 반영 |

## 발견된 이슈

| 마이 화면 `프로필 수정` 버튼이 실제 편집 기능으로 연결되지 않음 | Major | 해결 완료 |
| 지도 핀에서 프로필 사진이 반영되지 않음 | Minor | 해결 완료 |

## 알려진 제한사항

- 지도 핀 프로필 사진은 현재 `profileImageBase64`를 Realtime DB 브로드캐스트에 포함하는 방식이다. 이미지 용량이 커지면 장기적으로는 썸네일 또는 URL 기반 구조를 검토해야 한다.
- 홈 `같이 들은 러너`는 최근 세션을 기준으로 보여주며, 세션 삭제 정책이나 장기 보존 정책은 이번 범위에서 별도 조정하지 않았다.
- 프로필 수정은 현재 닉네임/사진/나이/키/몸무게만 다루며, 성별 등 온보딩의 다른 입력 항목은 포함하지 않았다.
