## 구현 요약

홈, 마이, 친구, 친구 요청, 친구 프로필 화면의 초기 데이터 로딩 상태에 스켈레톤 UI를 적용했다. 데이터가 비어 있는 화면과 아직 불러오는 중인 화면을 분리해, 로딩 중 빈 상태 문구나 0값 통계가 먼저 보이는 문제를 줄였다.

공통 스켈레톤 컴포넌트는 `DesignSystem`에 추가해 화면별로 재사용하도록 구성했다.

## 구현된 기능 목록

- [x] 공통 `SkeletonBlock`, `SkeletonCircle`, `SkeletonRow` 추가
- [x] 홈 화면 초기 로딩 skeleton 적용
  - 이번 주 러닝
  - 최근 러닝
  - 같이 들은 러너
- [x] 마이 화면 초기 로딩 skeleton 적용
  - 통계 영역
  - 차트 영역
  - 최근 활동 목록
- [x] `MyViewModel` 로딩 상태 추가
- [x] 친구 화면 초기 로딩 skeleton 적용
  - 친구 목록
  - 추천 친구
  - 검색 결과
- [x] 친구 요청 풀스크린 skeleton 적용
- [x] 친구 프로필 skeleton 적용
  - 활동 문구
  - 통계
  - 최근 들은 노래
- [x] pull-to-refresh 중 기존 데이터가 있으면 기존 화면 유지

## 계획서 대비 변경 사항

| 항목 | 계획 | 실제 구현 | 사유 |

## 알려진 제한사항

- 스켈레톤은 초기 로딩 UX 개선 목적이며, 네트워크 실패 상태 전용 UI는 기존 오류/빈 상태 흐름을 따른다.
- 애니메이션은 SwiftUI 기본 도형 기반 shimmer로 구현했고, 별도 외부 라이브러리는 사용하지 않았다.
