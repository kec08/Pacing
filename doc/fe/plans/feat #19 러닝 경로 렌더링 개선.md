## 1. 목적 및 배경

현재 러닝 화면에서 운동을 시작하지 않아도 지도에 경로가 그려지는 문제가 있다.  
또한 러닝 중 경로 선이 메인 컬러 단색에 가깝게 보이고, GPS 좌표가 들어오는 간격에 따라 선이 뚝뚝 이어지는 느낌이 있어 실제 러닝 경험이 매끄럽지 않다.

이번 작업은 경로 기록 조건을 러닝 활성 상태로 제한하고, 지도 위 경로 표현을 Pacing 메인 컬러와 보라색 서브 컬러가 섞인 그라데이션으로 개선한다. 좌표 업데이트 시에는 화면상 경로가 더 자연스럽게 이어지도록 렌더링 방식을 보완한다.

추가로 온보딩에서 위치 권한을 승인한 사용자는 러닝을 시작하지 않아도 러닝 탭 첫 진입 시 즉시 현재 위치 중심 지도를 보아야 하며, 앱 사용 중 다른 탭으로 이동해도 내 위치 브로드캐스트가 유지되어 친구 지도에 계속 표시되어야 한다. 재생 중 음악은 러닝 화면에서 별도 현재 곡 UI로 바로 인지할 수 있어야 한다.

## 2. 사용자 시나리오

```
사용자가 러닝 화면에 진입한다
→ 아직 운동을 시작하지 않은 상태에서는 현재 위치만 보이고 경로는 표시되지 않는다
→ 온보딩에서 권한을 승인한 사용자는 러닝 탭 첫 진입 시 바로 현재 위치 중심으로 지도가 잡힌다
→ 러닝을 시작하면 그 시점부터 경로가 기록되고 지도에 표시된다
→ 이동할수록 경로가 메인 컬러와 보라색이 섞인 그라데이션으로 부드럽게 이어진다
→ 정지 버튼을 눌러 일시정지한 뒤 다시 시작해도 기존 경로 뒤에서 이어진다
→ 일시정지/종료 상태에서는 의도하지 않은 좌표가 경로에 추가되지 않는다
→ 사용자가 홈/친구/마이 탭으로 이동해도 앱 사용 중이면 내 위치 브로드캐스트는 유지된다
→ 음악을 재생 중이면 러닝 화면에서 현재 곡 UI를 바로 확인하고 음악 시트로 진입할 수 있다
```

## 3. 화면 구성

### 주요 화면
- 러닝 화면: 지도 위 현재 위치, 주변 러너, 러닝 경로 표시
- 러닝 완료 요약 화면: 저장된 경로 표시 유지

### UI 요소
- 러닝 경로 라인: 메인 컬러 + 보라색 서브 컬러 그라데이션
- 현재 위치 핀: 기존 스타일 유지

## 8. 특이 사항 / 기술 검토

- 현재 `LocationManager.locationManagerDidChangeAuthorization`에서 권한 승인 즉시 `startUpdatingLocation()`을 호출하고, `didUpdateLocations`에서 항상 `routeCoordinates.append`를 수행하는 구조다. 이 부분이 운동 전 경로 생성의 핵심 원인으로 보인다.
- 현재 위치 표시는 운동 전에도 필요하므로 위치 업데이트 자체를 막기보다, 경로 기록 여부를 별도 상태로 분리하는 방향이 안전하다.
- 위치 브로드캐스트를 `RunningView.onAppear/onDisappear`에 묶으면 탭 이동만으로도 `activeRunners`에서 내 노드가 사라진다. 앱 사용 중 위치 노출을 유지하려면 탭 루트 또는 앱 레벨 수명주기에서 관리해야 한다.
- `pause`/`resume`은 하나의 러닝 세션을 이어가는 동작이므로 경로를 초기화하지 않는다. 경로 초기화는 저장/폐기 후 `reset` 또는 새 러닝 시작 전 명시 초기화 시점에만 수행한다.
- `MapPolyline` 하나에 단일 그라데이션을 적용하면 지도 좌표 진행 방향과 화면 좌표 방향이 완전히 일치하지 않을 수 있다. 자연스러운 진행형 그라데이션이 필요하면 좌표 세그먼트 단위 렌더링을 검토한다.
- GPS 좌표 보간은 실제 운동 기록 데이터를 왜곡할 수 있으므로, 저장 데이터는 원본 좌표를 유지하고 화면 렌더링용 보정만 최소 적용한다.
- 최종 보고서, 최종 커밋, 푸시, PR은 사용자 QA 및 명시 승인 후 진행한다.
