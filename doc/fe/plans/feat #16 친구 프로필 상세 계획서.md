## 1. 목적 및 배경

친구 탭 MVP는 친구 목록, 추천 친구, 친구 요청 흐름까지 제공한다. 다음 단계에서는 친구 카드에서 친구의 러닝/음악 취향을 확인할 수 있는 상세 화면을 추가해 친구 관계가 단순 목록에서 끝나지 않고 같이 듣기와 공유 기능으로 자연스럽게 이어지도록 만든다.

이번 범위는 공유 탭 구현 전에 진행하는 친구 기능 세부화 작업이다. 사용자는 친구 카드를 탭해 해당 친구 프로필로 이동하고, 중앙 프로필 영역에서 친구의 기본 정보와 러닝 요약 지표를 확인하며, 아래 리스트에서 친구가 최근 들은 노래를 확인할 수 있어야 한다.

## 2. 사용자 시나리오

```
사용자가 친구 탭에 진입하면
→ 친구 목록의 친구 카드를 탭하고
→ 친구 프로필 상세 화면으로 이동한다
→ 중앙 프로필 영역에서 아바타/닉네임과 평균 페이스, 운동한 총 시간, 누적 km를 확인한다
→ 아래 최근 노래 리스트에서 친구가 최근 들은 음악을 확인한다
```

## 3. 화면 구성

### 주요 화면
- 친구 탭 메인: 친구 카드 탭 시 상세 화면으로 이동
- 친구 프로필 상세: 친구 아바타/닉네임, 러닝 요약 지표, 최근 들은 노래 리스트

### UI 요소
- 상단 네비게이션: 뒤로가기, 화면 제목 또는 친구 이름
- 중앙 프로필 영역:
  - 프로필 이미지 또는 이니셜 아바타
  - 닉네임
  - 최근 상태 텍스트
- 러닝 요약 지표:
  - 평균 페이스
  - 운동한 총 시간
  - 누적 km
- 최근 노래 리스트:
  - 노래 제목
  - 아티스트
  - 최근 청취/저장 시점
  - 앨범 아트는 현재 저장 데이터가 없으므로 1차 구현에서는 기본 음악 아이콘 사용
- 빈 상태:
  - 러닝 기록이 없을 때 통계는 `--` 또는 `0` 기반으로 표시
  - 최근 노래가 없을 때 박스형 과장 UI 없이 간결한 빈 상태 표시

## 4. 데이터 흐름

### 사용 데이터
- Firestore 컬렉션:
  - `users/{friendUID}`: 닉네임, 프로필 이미지, 상태 텍스트
  - `users/{friendUID}/runHistory`: 평균 페이스, 운동 총 시간, 누적 km 산출
  - `users/{friendUID}/recentSongs`: 최근 들은 노래 리스트 신규 저장 후보
- Realtime DB 경로:
  - `activeRunners/{uid}`: 현재 러닝 중인 노래 확인은 가능하지만, 최근 들은 노래 리스트 보존 용도로는 부적합
- MusicKit API:
  - 기존 러닝 화면의 현재곡 정보는 앱 내부에서 확인 가능
  - 친구 상세 화면에서는 친구의 MusicKit 라이브러리에 직접 접근하지 않고 서버에 저장된 공개 데이터만 표시

### 상태 관리
- ViewModel:
  - `FriendProfileViewModel` 신규 추가
- Published 프로퍼티:
  - `friend: FriendUser`
  - `stats: FriendProfileStats`
  - `recentSongs: [FriendRecentSong]`
  - `isLoading: Bool`
  - `errorMessage: String?`
- 신규 모델 후보:
  - `FriendProfileStats`
    - `averagePace: Double`
    - `totalDuration: Int`
    - `totalDistance: Double`
  - `FriendRecentSong`
    - `id: String`
    - `title: String`
    - `artistName: String`
    - `playedAt: Date?`
    - `songStoreID: String?`

## 8. 특이 사항 / 기술 검토

- 러닝 통계는 기존 `users/{uid}/runHistory` 데이터로 산출 가능하다.
- 현재 `RunRecord`에는 곡 정보가 저장되지 않는다. 최근 들은 노래 리스트를 안정적으로 보여주려면 `users/{uid}/recentSongs` 같은 별도 저장 구조가 필요하다.
- Realtime DB `activeRunners/{uid}`는 현재 러닝 중인 노래 확인에는 적합하지만, 러닝 종료 후 친구 상세에서 볼 최근 노래 기록으로는 적합하지 않다.
- 최근 노래 저장 시 개인정보/노출 범위를 고려해야 한다. 이번 범위에서는 친구에게 공개 가능한 최소 정보인 `title`, `artistName`, `songStoreID`, `playedAt`만 저장하는 방향을 우선 검토한다.
- 앨범 아트는 현재 저장 구조에 없으므로 1차 구현에서는 기본 음악 아이콘을 사용하고, MusicKit catalog 조회 또는 artwork URL 저장은 후속 개선으로 분리한다.
- 친구 요청 빈 상태 중앙 정렬은 feat #15에서 이미 반영되어 dev에 머지된 상태다. 이번 feat에서는 친구 상세 기능에 집중한다.
