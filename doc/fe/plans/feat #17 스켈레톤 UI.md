## 1. 목적 및 배경

홈, 마이, 친구 목록, 친구 프로필 화면에서 데이터 로딩 중 실제 빈 상태 UI가 짧게 노출되는 문제가 있다. 사용자는 데이터가 없는 것으로 오해할 수 있으므로, 초기 데이터 로딩 중에는 스켈레톤 UI를 먼저 보여주고 로딩 완료 후에만 실제 데이터 또는 빈 상태 안내를 보여주도록 개선한다.

## 2. 사용자 시나리오

```
사용자가 주요 데이터 화면에 진입하면
→ 초기 데이터 로딩 중 스켈레톤 UI를 확인하고
→ 로딩 완료 후 실제 데이터 또는 빈 상태 안내를 확인한다
```

## 3. 화면 구성

### 주요 화면
- 홈: 헤더, 이번 주 러닝 카드, 최근 러닝 리스트, 같이 들은 러너 영역
- 마이: 프로필 헤더, 통계/차트, 러닝 기록 영역
- 친구: 친구 목록, 추천 친구, 검색 결과 로딩 영역
- 친구 요청: 받은 요청 풀스크린 목록
- 친구 프로필: 프로필 헤더, 통계, 최근 들은 노래 리스트

### UI 요소
- `SkeletonBlock`: 둥근 사각형 placeholder
- `SkeletonCircle`: 프로필/아바타용 원형 placeholder
- `SkeletonRow`: 목록 row placeholder
- 화면별 skeleton section

## 4. 데이터 흐름

### 사용 데이터
- Firestore 컬렉션:
  - `users/{uid}/runHistory`
  - `users/{uid}/friends`
  - `users/{uid}/recentSongs`
  - `friendRequests`
- Realtime DB 경로: 없음
- MusicKit API: 없음

### 상태 관리
- `HomeViewModel`
  - 기존 `isLoading` 사용
  - 초기 로딩 중 빈 상태 대신 홈 skeleton 표시
- `MyViewModel`
  - `isLoading` 추가
  - `loadProfile`, `loadData` 로딩 흐름 분리 또는 초기 데이터 로딩 상태 관리
- `FriendsViewModel`
  - 기존 `isLoading`, `isSearching` 사용
  - 친구/추천/검색/요청 목록 skeleton 표시
- `FriendProfileViewModel`
  - 기존 `isLoading` 사용
  - 프로필/통계/최근 노래 skeleton 표시

## 6. 완료 기준 (Acceptance Criteria)

- [ ] 초기 로딩 중 홈 화면에 빈 상태 문구가 먼저 보이지 않는다.
- [ ] 초기 로딩 중 마이 화면에 0값 통계가 먼저 보이지 않는다.
- [ ] 초기 로딩 중 친구 목록/추천 친구에 빈 상태 문구가 먼저 보이지 않는다.
- [ ] 초기 로딩 중 친구 프로필에 `러닝 기록 없음`, `최근 들은 노래가 없어요`가 먼저 보이지 않는다.
- [ ] 로딩 완료 후 데이터가 없을 때만 빈 상태 UI가 표시된다.
- [ ] pull-to-refresh 중 기존 데이터가 있으면 기존 화면을 유지한다.
- [ ] skeleton UI가 기존 메인 컬러/라이트 톤과 어울린다.
- [ ] Debug 빌드가 통과한다.

## 8. 특이 사항 / 기술 검토

- 최종 개발 보고서는 사용자 실기기 QA와 검토 이후 마지막에 작성한다.
- 이 계획서 승인 후 GitHub 이슈를 생성하고, `dev`에서 새 브랜치를 분기한다.
- 기존 AppIcon staged 변경은 이번 작업과 무관하므로 포함하지 않는다.
- 스켈레톤은 첫 진입 로딩에 집중하고, 이미 데이터가 있는 상태의 refresh에서는 기존 데이터를 유지한다.
- 스켈레톤은 별도 외부 라이브러리 없이 SwiftUI 기본 도형/애니메이션으로 구현한다.
