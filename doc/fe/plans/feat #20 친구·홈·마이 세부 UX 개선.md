## 1. 목적 및 배경

친구/홈/마이 화면은 기본 기능은 동작하지만, 실제 사용 흐름에서 정보 우선순위와 UI 일관성이 아직 맞지 않는다.  
현재 추천 친구는 친구 목록과 다른 카드형 가로 UI로 분리되어 있어 같은 성격의 데이터가 다른 방식으로 보이고, 홈의 `같이 들은 러너`는 데이터가 표시되지 않으며, 마이 페이지 헤더는 사용자에게 덜 중요한 신체 정보가 먼저 보여 프로필 상태 인지가 약하다.

이번 작업은 기존 구조를 크게 흔들지 않고, 친구 탭은 리스트 중심으로 정리하고, 홈은 실제 같이 들은 기록을 노출하며, 마이는 친구 프로필과 같은 상태 텍스트 중심으로 정리한다.  
추가로 마이의 `프로필 수정` 버튼은 실제 수정 가능한 편집 화면으로 연결해 프로필 사진/닉네임/키/몸무게/나이를 앱 내에서 갱신할 수 있도록 한다.

## 2. 사용자 시나리오

```
사용자가 친구 탭에 진입하면
→ 추천 친구도 기존 친구 목록처럼 세로 리스트로 확인하고
→ 각 행 오른쪽의 추가 버튼으로 바로 친구 요청을 보낸다
→ 이름 아래에는 "어제 러닝 완료" 같은 상태 텍스트를 본다

사용자가 홈 탭에 진입하면
→ 같이 들은 러너 목록에서 상대 닉네임과 같이 들은 시점을 확인한다

사용자가 마이 탭에 진입하면
→ 프로필 아래에서 나이/키/몸무게 대신 현재 운동 상태를 확인하고
→ 프로필 수정 버튼을 눌러 사진, 이름, 키, 몸무게, 나이를 수정한다
```

## 3. 화면 구성

### 주요 화면
- 친구 탭 메인: 친구 목록 + 추천 친구 리스트 통일
- 홈 탭: 같이 들은 러너 섹션 데이터 연결 및 표시 문구 보강
- 마이 탭 메인: 프로필 헤더 상태 텍스트 노출
- 프로필 수정 화면: 사진/닉네임/신체 정보 수정 폼

### UI 요소
- 추천 친구 리스트 행:
  - 친구 행과 동일한 세로형 레이아웃
  - 좌측 아바타
  - 중앙 닉네임 + 상태 텍스트
  - 우측 `추가` 버튼
- 같이 들은 러너 행:
  - 상대 닉네임
  - 같이 들은 시점 또는 상대 시간 정보
  - 필요 시 현재 곡명/아티스트 보조 정보 유지
- 마이 프로필 헤더:
  - 프로필 이미지
  - 닉네임
  - 친구 프로필과 동일한 활동 상태 텍스트
- 프로필 수정 화면:
  - 프로필 사진 선택
  - 닉네임 입력
  - 나이 선택
  - 키/몸무게 선택
  - 저장 버튼

## 4. 데이터 흐름

### 사용 데이터
- Firestore 컬렉션:
  - `users/{uid}`: 닉네임, 프로필 이미지, 키, 몸무게, 나이
  - `users/{uid}/friends/{friendUID}`: 친구 목록/상태 텍스트
  - `users/{uid}/runHistory`: 최근 러닝 날짜 기반 상태 텍스트 계산
- Realtime DB 경로:
  - `listenSessions/{sessionID}`: 같이 듣기 세션 정보
- PhotosPicker:
  - 프로필 이미지 재선택

### 상태 관리
- ViewModel:
  - `FriendsViewModel`
  - `HomeViewModel`
  - `MyViewModel`
  - `ProfileEditViewModel` 신규 또는 `MyViewModel` 확장 검토
- Published 프로퍼티:
  - `recommendedUsers`
  - `recentListenSessions`
  - `nickname`, `profileImage`, `age`, `height`, `weight`
  - `activityStatusText` 신규 필요 가능
  - 프로필 수정 시트/화면 표시 상태

## 8. 특이 사항 / 기술 검토

- 현재 친구 목록은 `users/{uid}/friends/{friendUID}` 문서에서 가져오며, 추천 친구는 `users` 최신 생성 순으로 조회한다. 추천 친구에 친구와 동일한 상태 텍스트를 안정적으로 보여주려면 `users.statusText` 품질을 먼저 점검하거나, 부족하면 최근 러닝 날짜 기반 fallback을 추가해야 한다.
- `HomeViewModel`은 현재 `recentListenSessions`를 항상 빈 배열로 두고 있어 UI만 있고 데이터 로딩이 없는 상태다. 실제 세션 기록을 표시하려면 Realtime DB `listenSessions` 조회 API를 추가해야 한다.
- 현재 `ListenSession.partnerNickname`은 항상 `guestNickname`을 반환하므로, 현재 사용자가 guest인 경우에는 잘못된 상대 이름이 표시될 수 있다. 홈 섹션 구현 시 이 편의 프로퍼티는 사용자 기준으로 다시 정의해야 한다.
- 마이 화면의 `프로필 수정`은 현재 버튼만 있고 동작이 없다. 온보딩의 `ProfileSetupView`는 단계형 최초 입력 플로우라서, 그대로 재사용하기보다 편집 전용 뷰/폼으로 분리하는 편이 유지보수에 유리하다.
- 프로필 수정 저장 시 Firestore와 UserDefaults를 함께 갱신해야 로그인 직후/앱 재실행 시 값이 일관되게 보인다.
- 이번 작업은 세부 UX 개선 범위이므로, 친구 관계 정책이나 같이 듣기 세션 구조 자체를 크게 바꾸는 리팩토링은 하지 않는다.
