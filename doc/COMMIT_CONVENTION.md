# 커밋 메시지 규칙 (Commit Convention)

## 기본 형식

```
타입: 제목 (#이슈번호)

본문 (선택)

푸터 (선택)
```

## 타입 목록

| 타입 | 용도 |
|------|------|
| `feat` | 새로운 기능 추가 |
| `fix` | 버그 수정 |
| `ui` | UI/레이아웃 변경 (기능 변경 없음) |
| `refactor` | 코드 리팩터링 (기능 변경 없음) |
| `test` | 테스트 코드 추가/수정 |
| `docs` | 문서 추가/수정 |
| `chore` | 빌드 설정, 패키지 관리 등 |
| `hotfix` | 프로덕션 긴급 버그 수정 |

## 제목 규칙

- 한국어로 작성
- 50자 이내
- 마침표 없음
- 명령형으로 작성 (예: "추가", "수정", "삭제")

## 예시

```bash
# 기능 추가
feat: MapKit 지도 화면 기본 구성 (#3)

# 버그 수정
fix: 러닝 종료 후 타이머 멈추지 않는 문제 수정 (#14)

# UI 변경
ui: 주변 러너 핀 버블 디자인 조정

# 리팩터링
refactor: LocationManager 싱글톤 패턴으로 변경

# 문서
docs: 브랜치 전략 문서 추가

# 여러 줄 커밋 (본문 포함)
feat: Firebase Realtime DB 위치 브로드캐스트 구현 (#7)

- 5초 간격으로 위치 업데이트 전송
- 앱 백그라운드 진입 시 브로드캐스트 유지
- 러닝 종료 시 activeRunners에서 제거
```

## 나쁜 예시

```bash
# ❌ 너무 모호함
fix: 수정

# ❌ 여러 기능을 한 커밋에
feat: 지도 추가, 음악 연동, 러너 핀 표시

# ❌ 영어/한국어 혼용
feat: Add 러닝 map screen

# ❌ 마침표
feat: 러닝 탭 기본 화면 구성.
```

## 브랜치별 커밋 전략

### feat 브랜치
- 작업 단위를 작게 쪼개어 커밋
- 하나의 커밋 = 하나의 완성된 변경
- WIP(작업 중) 커밋은 push 전 squash

### fix 브랜치
- 수정 원인과 방법을 본문에 명시
- 관련 이슈 번호 반드시 포함

### 최종 PR 전 정리
- `git rebase -i` 로 불필요한 커밋 정리
- 의미 있는 단위로 squash
