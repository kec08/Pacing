1. 프로젝트 개요

1.1 앱 소개
Pacing은 러닝 중 근처에 있는 사람과 Apple Music 플레이리스트를 실시간으로 동기화하여 같이 들을 수 있는 iOS 러닝 앱입니다. 지도 위에서 주변 러너를 발견하고, 그 사람이 듣는 음악을 확인한 뒤 '같이 듣기'로 연결되는 새로운 러닝 경험을 제공합니다. 친구 네트워크와 플레이리스트 공유를 통해 러너 커뮤니티를 형성합니다.

1.2 핵심 가치
	•	혼자 달리지만 혼자가 아닌 느낌 — 음악으로 연결되는 러너 커뮤니티
	•	같은 비트, 같은 페이스 — 음악 싱크를 통한 간접적 동반 러닝
	•	발견의 즐거움 — 지도 위 낯선 러너의 플레이리스트 탐색
	•	연결의 즐거움 — 친구와 함께 달리는 경험, 플레이리스트로 이어지는 관계

1.3 타깃 사용자
	•	Apple Music 구독 중인 20~30대 러너
	•	혼자 러닝하지만 음악으로 동기부여가 필요한 사용자
	•	새로운 음악을 발견하고 싶은 러너
	•	함께 달릴 러닝 메이트를 찾는 사용자

4. 화면 구성 및 플로우

4.1 홈 탭
화면 구성
	•	상단: 인사 텍스트 + 오늘 날짜/날씨
	•	이번 주 러닝 통계 카드 — 총 거리 / 총 시간 / 평균 페이스
	•	최근 러닝 기록 리스트 — 날짜, 거리, 시간, 경로 썸네일
	•	최근 같이 들은 사람 섹션 — 아바타 + 닉네임 + 함께 들은 곡

데이터 소스
	•	Firestore runHistory 컬렉션에서 최근 7일 기록 조회
	•	listenSessions 컬렉션에서 최근 같이 들은 세션 조회

4.2 친구 탭 (신규 — v1.2)
화면 구성
	•	상단 검색바 — 닉네임 검색으로 유저 찾기
	•	친구 요청 섹션 — 받은 요청 수락/거절
	•	내 친구 목록 — 아바타 + 닉네임 + 최근 러닝 상태
	•	추천 친구 섹션 — 우선순위: 연락처 일치 > 가까운 러너 > 랜덤

친구 추천 우선순위
	•	1순위 연락처 기반: CNContactStore로 전화번호 수집 → SHA256 해시 → Firestore users 컬렉션과 매칭
	•	2순위 가까운 러너: Realtime DB activeRunners에서 현재 1km 반경 내 러너
	•	3순위 랜덤: 최근 7일 내 활동 있는 유저 중 랜덤 추천 (최대 10명)

친구 요청 플로우
	•	추천 카드에서 "친구 추가" 탭 → Firestore friendRequests 문서 생성 (status: pending)
	•	상대방 친구 탭 진입 시 "받은 요청" 섹션에 표시
	•	수락 → 양방향 friends 컬렉션에 추가 / 거절 → friendRequests 문서 삭제

4.3 러닝 탭
기본 화면 구성
	•	전체 화면 MapKit 지도
	•	내 위치 중심 표시
	•	주변 러너 핀 — 아바타 + 현재 듣는 곡 버블 실시간 표시
	•	화면 중앙 하단: 대형 시작 버튼
	•	시작 버튼 좌우: 음악 버튼 / 주변 찾기 버튼

시작 버튼 동작
	•	러닝 시작: CoreLocation 고정밀 수집 시작 → 지도에 Polyline 실시간 드로잉 → 상단에 거리/시간 실시간 표시
	•	러닝 중 시작 버튼: 일시정지 버튼으로 전환
	•	종료: 거리/시간/경로 요약 화면 → Firestore 저장

음악 버튼 (Bottom Sheet)
	•	현재 재생 중인 곡 — 앨범아트 / 곡명 / 아티스트
	•	재생/일시정지, 이전/다음 곡 컨트롤
	•	MusicKit ApplicationMusicPlayer 연동

주변 찾기 버튼 (Bottom Sheet)
	•	필터: 친구 / 가까운 러너
	•	러너 카드: 아바타 | 닉네임 | 현재 곡명 | 거리 | 같이 듣기 버튼
	•	같이 듣기 버튼 탭 → 연결 요청 전송
	•	상대방 수락 → 싱크 세션 시작 + 상단 배너 표시

같이 듣기 싱크 플로우
	•	1단계: 호스트가 '같이 듣기' 요청 → Realtime DB 세션 생성
	•	2단계: 게스트에게 알림 → 수락/거절
	•	3단계: 수락 시 곡 ID + 재생 위치 + 타임스탬프 전송
	•	4단계: 게스트 기기에서 동일 곡 동일 위치로 재생 시작
	•	5단계: 재생/일시정지/곡 변경 이벤트 실시간 브로드캐스트
	•	6단계: 세션 종료 시 각자 독립 재생 복귀
※ 싱크 오차: 네트워크 레이턴시로 인해 100~300ms 내외 허용 (MVP 기준)

4.4 공유 탭 (신규 — v1.2)
화면 구성
	•	상단 탭: 탐색 / 내 공유
	•	탐색 탭: 다른 유저가 공유한 러닝 플레이리스트 피드
	  - 카드: 프로필 아바타 + 닉네임 + 플레이리스트명 + 트랙 수 + 총 재생시간
	  - 정렬: 최신순 / 인기순 (저장 수 기준)
	  - "재생하기" 버튼 → Apple Music으로 해당 플레이리스트 재생
	  - "저장하기" 버튼 → 내 Apple Music 라이브러리에 추가
	•	내 공유 탭: 내가 공유한 플레이리스트 목록
	  - "공유하기" 버튼 → 내 Apple Music 플레이리스트 선택 → Firestore 등록
	  - 공개/비공개 토글
	  - 공유 취소 (Firestore에서 삭제)

공유 플레이리스트 플로우
	•	"공유하기" 탭 → MusicKit으로 내 플레이리스트 목록 불러오기
	•	플레이리스트 선택 → 제목/설명 입력 → Firestore sharedPlaylists 문서 생성
	•	탐색 피드에 실시간 노출
	•	다른 유저가 "재생하기" → MusicKit으로 해당 플레이리스트 ID 전달 → 재생
	•	다른 유저가 "저장하기" → Apple Music API로 라이브러리에 추가

4.5 마이 탭
신체 정보 섹션
	•	프로필 이미지 / 닉네임 / 가입일
	•	키 / 체중 / 나이 / 성별 입력 및 수정
	•	HealthKit 연동 여부 설정 (선택)

활동 통계 섹션
	•	이번 달 총 러닝 거리 / 횟수 / 평균 페이스
	•	월별 러닝 그래프 (막대 차트)
	•	최고 기록 — 최장 거리 / 최고 페이스

최근 활동 섹션
	•	최근 러닝 기록 리스트 5개 미리보기 + 더보기
	•	기록 탭 → 상세 화면 (경로 지도 + 구간별 페이스)

설정 섹션
	•	탐색 반경 기본값 설정
	•	Apple Music 연동 상태 확인
	•	알림 설정 (같이 듣기 요청 / 친구 요청 알림)
	•	로그아웃 / 계정 탈퇴


5. 주요 사용자 플로우

5.1 온보딩 플로우
	•	앱 최초 실행 → Apple 로그인 (Firebase Auth)
	•	위치 권한 요청 (항상 허용 권장)
	•	Apple Music 권한 요청 (MusicKit)
	•	연락처 권한 요청 (친구 추천용, 선택)
	•	닉네임 + 신체 정보 입력
	•	연락처 기반 친구 추천 화면 (건너뛰기 가능) → 메인 화면 진입

5.2 러닝 시작 플로우
	•	러닝 탭 진입 → 지도 로드, 주변 러너 핀 표시
	•	중앙 시작 버튼 탭
	•	CoreLocation 고정밀 수집 시작
	•	Realtime DB에 내 위치 + 현재 곡 정보 브로드캐스트 (5초 간격)
	•	지도에 이동 경로 Polyline 실시간 드로잉
	•	러닝 종료 → 요약 화면 → Firestore 저장 → 홈 탭에 반영

5.3 같이 듣기 플로우
	•	주변 찾기 버튼 → Bottom Sheet 오픈
	•	원하는 러너의 '같이 듣기' 버튼 탭
	•	상대방 수락 → 싱크 세션 시작
	•	화면 상단에 '지금 함께 듣는 중: [닉네임]' 배너 표시
	•	한쪽이 종료하면 세션 해제, 각자 독립 재생

5.4 친구 추가 플로우
	•	친구 탭 진입 → 연락처 권한 확인
	•	연락처 있으면 전화번호 해시 매칭으로 추천 목록 표시
	•	"친구 추가" 탭 → 상대방에게 요청 전송
	•	상대방 수락 → 친구 목록에 추가
	•	닉네임 검색으로 직접 찾기 가능

5.5 플레이리스트 공유 플로우
	•	공유 탭 → "내 공유" → "공유하기" 탭
	•	내 Apple Music 플레이리스트 선택
	•	제목/설명 입력 후 공개 설정
	•	Firestore에 등록 → 탐색 피드에 노출
	•	다른 유저가 재생 또는 저장 가능
